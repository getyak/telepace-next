# 10 个 AI 交互创建问卷的用户故事（Creation-Stage）

> **与 `docs/scenarios.md` 的正交关系**：
> - `docs/scenarios.md` 的 S1–S10 关注**端到端**（build → dispatch → interview → insight）跑通 + 12 维 rubric。

## US-01 — 独立咖啡店老板测新品牌
**角色**：三十出头的独立精品咖啡店主，非营销出身，第一次做用户研究
**场景一句话**："我想换个 logo，但不知道熟客会不会喜欢。"

**AI 对话剧本**：
1. 用户："我想调研老顾客对我新 logo 的看法。"
2. AI（智能感）：识别为 **Creative Testing**；反问"新 logo 是完全重塑还是微调？给我看下老 logo 或者描述一下。" + 主动假设："我猜你担心的是老顾客会觉得失去了原有的温暖感，对吗？我把这个假设加进研究目标。"
3. 用户："对，就是怕失去温暖感。"
4. AI：**推断目标人群**："我会重点招募过去 90 天到店 3 次以上的老客。你店在哪个城市？我加个地理筛选题。"
5. 用户："成都。"
6. AI：生成 6 题 outline（暖启动 → 品牌感知 → 新旧对比 → 情感锚点 → 推荐意愿 → 开放建议），并**同屏预览一位假想受访者的回答**。
7. 用户："第 4 题问得太直白了。"
8. AI：**当场改题** + 解释："我把它换成一个故事化情境："想象你带朋友第一次来店，看到新 logo 会怎么跟朋友介绍我们？""

**产出 CampaignSpec 字段**：`goal / hypotheses[温暖感] / target_persona / audience_screener[到店频次+地理] / outline.items[6 题带 goal 与 branch] / channels[web_text] / languages[zh]`

**审美/体验亮点**：右侧问卷卡片每题旁一枚**目标 chip**（暖启动/品牌感知/推荐意愿）；底部抽屉 **"AI 假想受访者试跑"**（头像+波形+文字气泡，可"再来一位"换 persona）。

**超越 Listen Labs 的关键**：Listen Labs 的 discussion guide 是"单向输出"；我们在**同屏**做到"生成→假想试跑→即时改题"闭环。

## US-02 — B2B SaaS PM 做用户流失分析
**角色**：SaaS 公司 Product Manager，订阅取消率上涨
**场景一句话**："我要采访 20 位近 30 天流失用户，找出他们真正的取消原因。"

**AI 对话剧本**：
1. 用户："我要访谈近 30 天流失的付费用户，20 位。"
2. AI：识别为 **Churn / Journey Analysis**；追问"你希望覆盖哪几个套餐层级？有没有你已经怀疑的原因？"
3. 用户："Basic 和 Pro，我怀疑是竞品降价。"
4. AI：把"竞品价格"设为**假设 A**，并**主动追加对照假设 B、C**："我加两个对照：B) 使用门槛太高，C) 关键功能不满足。这样可以避免只印证一个答案。"
5. 用户："好。"
6. AI 生成 outline，含**分支逻辑**："如果用户提到价格 → 追问替代方案定价；如果提到功能 → 追问具体工作流。"
7. AI：**主动给激励与时长建议**："15 分钟 / 25 USD 视频访谈是 Churn 场景甜点，我按 20 × 25 = 500 USD 建了草稿。"

**产出**：`hypotheses[3 条对照] / target_persona / budget_usd=500 / target_completions=20 / outline.items[8 题带 branch_if_positive/negative]`

**审美亮点**：假设卡片**三色 chip**（当前 / 对照 / 用户提出）；outline 分支用**折叠树** + spring 展开动效。

**超越点**：AI **主动补对照假设**防 confirmation bias（Listen Labs 只被动响应）。

## US-03 — UX 研究员做多市场原型可用性测试
**角色**：跨国零售品牌 UX 研究员
**场景一句话**："我要在 3 个国家测 Figma 原型的可用性，各 15 人。"

**AI 对话剧本**：
1. 用户："我要在美国、日本、巴西各招 15 人测这个 Figma 原型。"（拖入 Figma 链接）
2. AI：**识别 Figma 链接** + 自动截取 3 关键屏 + "我看到三个主流程：注册/搜索/结账，你想每个流程都测吗，还是只聚焦？"
3. 用户："聚焦搜索和结账。"
4. AI：为**每国**生成**本地化 outline**："日本受访者对隐私偏向委婉，第 5 题我改用'你会如何跟朋友描述这个体验'的间接问法。"
5. 用户："巴西呢？"
6. AI："巴西加一题移动网络稳定性，因为 3G 用户占比高。"

**产出**：`languages=[en, ja, pt-BR] / target_persona / outline.items[每国 6-8 题·本地化措辞] / channels=[web_voice]`

**审美亮点**：outline **三栏并列** + 国旗顶栏；每题右上**文化 tag**（"日本 · 间接措辞"）。

**超越点**：Listen Labs 的多语言是"翻译"；我们做**文化本地化 rewrite**，AI 显式说 why。

## US-04 — 品牌营销测新广告 slogan
**角色**：新消费品牌 Brand Marketer
**场景一句话**："3 个候选 slogan，想知道哪个最打动 Z 世代。"

**AI 对话剧本**：
1. 用户："三个候选：A / B / C。目标是 Z 世代。"
2. AI：**为三个 slogan 生成对照式 outline**（盲测，每人各触发一次投射式问题）
3. AI："我建议不要直接问'哪个最好'，而是让受访者**给每条 slogan 起一个人物名字**——投射测试，暴露真实情感反应。愿意试试吗？"
4. 用户："这个思路好。"
5. AI：**同屏演示**假想 Z 世代回答（含 emoji 情绪标签），右上标注"情感强度预测：8.2/10"。

**产出**：`hypotheses[3 条对照] / outline.items[3 组循环 + 尾部对比题] / insight kind=[verbatim + theme]`

**审美亮点**：outline 卡片**双击放大**成"单题全屏预览"；情感强度**渐变条**（冷灰→亮蓝）。

**超越点**：引入**投射测试方法学**（AI 教研究者用更专业问法），显示"AI 有研究学识"。

## US-05 — 医疗健康 App 做隐私敏感度调研
**角色**：数字健康 App 产品负责人
**场景一句话**："我要了解糖尿病用户对生物数据分享的接受度。"

**AI 对话剧本**：
1. 用户："调研 II 型糖尿病用户，愿不愿意分享血糖数据给第三方。"
2. AI：**主动提示合规**："这是 HIPAA/GDPR 敏感话题，我给每题加显式知情同意，敏感题标红可选跳过。同意吗？"
3. 用户："同意。"
4. AI：outline 中第 3-6 题带**红色 sensitivity dot** + "受访者可跳过"标记
5. AI：**招募筛选题**主动加"过去 30 天是否用 CGM 设备"，"CGM 用户对数据分享心智模型和指血用户很不同。"

**产出**：`audience_screener[医疗筛选] / outline.items[8 题带 sensitivity level] / channels=[web_text, web_voice]`

**审美亮点**：敏感题**红色圆点 + 微弱脉冲呼吸动效**；顶部**"合规守护"徽章** + 提示语。

**超越点**：**主动合规护栏**（Listen Labs 只在页脚提 SOC2 + GDPR，不进入产品体验）。

## US-06 — 早期创业者做 PMF 探索访谈
**角色**：种子轮创业者，还没有产品
**场景一句话**："想通过访谈验证：小微企业是否愿意为 AI 报税助手付费。"

**AI 对话剧本**：
1. 用户："我在做 AI 报税，想访谈潜在客户。"
2. AI：**识别为 PMF 探索**（非产品反馈）："这个阶段 Mom Test 建议不要问'你会用吗'，而是问'你上次报税花了多少时间？痛在哪？'。基于 Mom Test 帮你生成 outline，行吗？"
3. 用户："好。"
4. AI：**生成 outline**，每题下方注 Mom Test 原则（"谈过去，别谈未来"、"具体细节，别泛化"）
5. AI："建议 target_completions 先设 8 人，先出定性 → 再决定要不要 quant。省钱。"

**产出**：`outline.items[6 题 Mom Test 风格] / target_completions=8 / success_criteria=[≥5 人具体痛点, ≥3 人价格锚点]`

**审美亮点**：outline 每题下**方法学 tag**（"Mom Test · 谈过去"）；右上角**"研究方法学"徽章**可点开看引用。

**超越点**：**方法学显性化**（AI 有明确研究哲学，不是万能助手）。

## US-07 — 教育机构做课程反馈
**角色**：K12 在线教育课程主管
**场景一句话**："秋季课上完了，想同时听家长和孩子的反馈。"

**AI 对话剧本**：
1. 用户："同时访谈家长和 10-14 岁孩子，各 30 人。"
2. AI："孩子措辞和时长（<8 分钟）要调整。你要**两条独立 outline** 还是**共享主干+差异化措辞**？"
3. 用户："共享主干+差异化措辞。"
4. AI："好。生成主干 6 题 + 家长版 + 孩子版（含 emoji 选项与语气词），孩子版加 minor consent 提示。"
5. AI："孩子访谈建议用**语音**，文字打字对这个年龄段负担大。"

**产出**：`audience_screener[家长/孩子分流] / outline.items[6 主干 × 2 措辞版] / channels={家长: web_text, 孩子: web_voice}`

**审美亮点**：outline **并列双列**"家长版 / 孩子版"，滚动同步；孩子版气泡轻微**手绘风描边**。

**超越点**：**双身份共享主干**是方法学巧解，Listen Labs 没有原生支持。

## US-08 — 政府/公共政策做小样本深度访谈
**角色**：地方政府政策研究员，做老年数字鸿沟调研
**场景一句话**："采访 12 位不同城市的老年人，聊数字支付。"

**AI 对话剧本**：
1. 用户："采访 12 位 65 岁以上老人，关于数字支付。"
2. AI："这个人群更适合**电话或线下**而不是线上聊天。走电话？"（识别 `phone_outbound`）
3. 用户："电话，好。"
4. AI："outline 按低文化水平受访者优化：短句、无术语、多情境描述。加了一段温暖开场语（母语），以及结尾感谢与转介绍语。"
5. AI："我把 12 人分层：4 城市 × 3 层次（独居/与子女住/养老院），加一题'是否会用微信'的筛选？"

**产出**：`channels=[phone_outbound] / target_persona / outline.items[5 题·每题<30 字·无术语] / audience_screener[城市+生活状况+微信使用]`

**审美亮点**：电话渠道时右侧显示**音频波形 + 时长条**；温暖开场语可**朗读试听**（语音合成）。

**超越点**：**渠道语义化**：AI 根据人群主动推荐渠道，而不是让用户先选渠道。

## US-09 — 招聘/HR 做候选人体验回访
**角色**：大公司 HR，改善 Candidate Experience
**场景一句话**："访谈过去 60 天面试失败的候选人，找体验痛点。"

**AI 对话剧本**：
1. 用户："采访过去 60 天面试后被拒的候选人。"
2. AI："这个人群**心理防御强**，建议：1) 第三方匿名承诺；2) 头两题必须是感谢参与+可随时退出；3) 不问'你觉得我们哪里做得差'，而是'你在跟朋友描述这段面试时会怎么讲'。要按这个思路？"
3. 用户："对。"
4. AI：outline 首尾加**情绪缓冲题**，中间用**投射式 + 关键事件法（CIT）**
5. AI："我给这份研究打了一个**方法学套装 tag：Anonymous · Projective · CIT**，评分 agent 会按这套标准打分。"

**产出**：`outline.items[8 题·首尾缓冲·中段 CIT] / success_criteria=[≥4 人具体情境, ≥2 人改进建议]`

**审美亮点**：顶部**方法学套装徽章**（Anonymous / Projective / CIT，三枚 chip 组合）；缓冲题**低饱和柔和色**与主题题区分。

**超越点**：**方法学组合**是可复用资产（第二次做类似研究可一键调用）。

## US-10 — 学者做定性研究（配方法学严谨性）
**角色**：社会学博士候选人，做零工经济骑手心理调研
**场景一句话**："做 20 人半结构化访谈，写进论文。"

**AI 对话剧本**：
1. 用户："半结构化访谈，20 位外卖骑手，主题是自我认同。"
2. AI："学术场景我会更保守：1) 显式知情同意与录音授权；2) outline 只做'访谈提纲级别'（宽泛开放式）；3) 提供 **IPA (Interpretative Phenomenological Analysis)** 或 **主题分析** 方法学选项；4) 数据可导出 CAQDAS 兼容格式。选哪种？"
3. 用户："IPA。"
4. AI："IPA 强调深度而非广度，target_completions 建议从 20 调到 8-10。"
5. 用户："改成 10。"
6. AI："outline 按 IPA 六阶段：日常场景 → 关键转折 → 意义建构 → 身份词汇 → 关系他者 → 未来投射。"

**产出**：`target_completions=10 / outline.items[6 题 IPA 六阶段] / success_criteria=[每人 ≥30 分钟, ≥5 人身份词汇] / channels=[web_voice]`

**审美亮点**：outline **手稿风字体**（Serif 变体）暗示学术；顶部**"IPA 六阶段"进度环**（6 段弧线）；导出支持 **NVivo / MAXQDA** 格式。

**超越点**：**为学术定制的严谨姿态**（Listen Labs 是企业营销血统，学术是空白）。
